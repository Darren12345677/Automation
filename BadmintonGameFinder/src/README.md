# 🏸 Badminton Game Finder

A Telegram userbot that monitors badminton social group chats and alerts you to games in **West Singapore**. Runs as your own Telegram account using Telethon — no bot token needed, no API costs.

---

## How It Works

1. Logs in as your Telegram account via Telethon
2. Listens to messages in the groups defined in `config.py`
3. For groups with topics (forums), only listens to the topics you specify
4. Checks each message against badminton signal keywords, then West SG venue keywords
5. If both match, sends a formatted alert to your private channel

---

## Prerequisites

- Python 3.10+
- A Telegram account
- Telegram API credentials from [my.telegram.org/apps](https://my.telegram.org/apps)

## Local Setup

### 1. Clone and install dependencies

```bash
git clone <your-repo-url>
cd BadmintonGameFinder
pip install -r requirements.txt
```

### 2. Get your Telegram API credentials

1. Go to [my.telegram.org/apps](https://my.telegram.org/apps) and log in
2. Create an app — platform: **Desktop**, URL: leave blank
3. Copy the **App api_id** (short number) and **App api_hash** (long hex string)

### 3. Configure your environment

```bash
cp src/.env.example src/.env
```

Edit `src/.env`:

```
TELEGRAM_API_ID=12345678
TELEGRAM_API_HASH=abcdef1234567890abcdef1234567890
ALERT_CHAT=-1001234567890
```

For `ALERT_CHAT`, forward any message from your private channel to [@userinfobot](https://t.me/userinfobot) to get its numeric ID.

### 4. Configure groups and filters

Edit `src/config.py` to set which groups and topics to monitor, and which venues and keywords to filter on:

```python
WATCH_GROUPS = ["🏸 SG Badminton Community", "sg_badminton"]
WATCH_TOPICS = ["Available games", "Looking for Games/Game requests"]
```

### 5. First run

```bash
cd src
python main.py
```

On first run, Telethon will prompt for your phone number and a one-time OTP from Telegram. After login, a `badminton_session.session` file is saved — subsequent runs skip the login step.

## Railway Deployment

Since Railway has no interactive terminal, you need to convert your local session to a string and pass it as an environment variable.

### 1. Generate a session string locally

After completing the first local run above, run this once:

```bash
cd src
python generate_session.py
```

Copy the printed session string.

### 2. Set environment variables on Railway

In your Railway project → **Variables**, add:

| Variable | Value |
|---|---|
| `TELEGRAM_API_ID` | your api_id |
| `TELEGRAM_API_HASH` | your api_hash |
| `ALERT_CHAT` | your channel ID |
| `TELEGRAM_SESSION` | the string from step 1 |

### 3. Deploy

Push to your connected GitHub repo. Railway will detect Python, install dependencies from `requirements.txt`, and start the app using the `Procfile`.

> **Note:** `badminton_session.session` is gitignored. The bot uses `TELEGRAM_SESSION` env var on Railway and falls back to the local session file when running locally.

## Config Reference

### Environment variables (`.env`)

| Variable | Required | Description |
|---|---|---|
| `TELEGRAM_API_ID` | ✅ | From my.telegram.org |
| `TELEGRAM_API_HASH` | ✅ | From my.telegram.org |
| `ALERT_CHAT` | ✅ | Where to send alerts — `me`, `@username`, or numeric channel ID |
| `TELEGRAM_SESSION` | Railway only | Session string generated by `generate_session.py` |

### Application settings (`config.py`)

| Setting | Description |
|---|---|
| `WATCH_GROUPS` | Group titles or `@usernames` to monitor. Empty = all groups. |
| `WATCH_TOPICS` | Forum topic names to listen to. Empty = all topics. |
| `WEST_VENUES` | Lowercase venue keywords for West SG detection. Add any missing venues here. |
| `BADMINTON_SIGNALS` | Keywords that indicate a message is a game post. |

## Troubleshooting

**Bot not detecting messages from a group**
→ Check that the group name or username exactly matches an entry in `WATCH_GROUPS` in `config.py`. Usernames are matched without `@`.

**West SG games being missed**
→ Check the terminal output — if the venue isn't matching, add its name to `WEST_VENUES` in `config.py`.

**Railway deployment: `ValueError: Cannot find any entity`**
→ Make sure `ALERT_CHAT` is set as an integer (not a string) in Railway's variables, or verify the channel ID is correct.

**Session expired on Railway**
→ Re-run `generate_session.py` locally and update the `TELEGRAM_SESSION` variable in Railway.
